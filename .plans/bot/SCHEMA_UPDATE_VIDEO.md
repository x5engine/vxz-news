## Required Schema Changes

### 1. Media Links Enhancement (IMMEDIATE)

**Current Schema:**
```json
{
  "media_links": [
    {
      "source": "Reuters/AP",
      "url": "https://news.yahoo.com/article-link",
      "media": "https://media.zenfs.com/image.jpg"
    }
  ]
}
```

**Updated Schema:**
```json
{
  "media_links": [
    {
      "source": "Reuters/AP",
      "url": "https://news.yahoo.com/article-link",
      "media": "https://media.zenfs.com/video.mp4",
      "thumbnail": "https://media.zenfs.com/thumbnail.jpg",  // NEW - Optional
      "duration": 120,                                        // NEW - Video duration in seconds
      "format": "mp4",                                        // NEW - File format (mp4, webm, youtube, jpg, png)
      "type": "source_video"                                  // NEW - Type: "source_video", "source_image", "ai_generated"
    }
  ]
}
```

### New Fields Explanation:
- **`thumbnail`** (String, Optional): Direct URL to video thumbnail image. If not provided, frontend will use YouTube API or generate from video frame.
- **`duration`** (Number, Optional): Video length in seconds. Used for UI duration overlay (e.g., "2:30").
- **`format`** (String, Required): Media format identifier. Supports: `mp4`, `webm`, `youtube`, `jpg`, `png`, `gif`.
- **`type`** (String, Required): Media source type:
  - `"source_video"` - Video scraped from news article
  - `"source_image"` - Image scraped from news article
  - `"ai_generated"` - Video/audio generated by VXZ AI pipeline (Phase 2)

## Backend Scraper Updates

### A. Video Extraction Enhancement
**Current Issue:** Most events lack video content, limiting TV Mode engagement.

**Action Required:**
1. **Aggressive Video Scraping:**
   - Scan article HTML for `<video>` tags
   - Extract YouTube/Vimeo embeds from `<iframe>` tags
   - Parse Twitter/X video embeds
   - Check for HTML5 video sources (`<source src="...">`)

2. **Video Metadata Extraction:**
   ```javascript
   // Pseudo-code for backend scraper
   const videoElement = dom.querySelector('video, iframe[src*="youtube"]');
   if (videoElement) {
     const videoUrl = extractVideoUrl(videoElement);
     const duration = await getVideoDuration(videoUrl); // Use yt-dlp or video-metadata library
     const thumbnail = await extractThumbnail(videoUrl);
     
     mediaLinks.push({
       source: articleSource,
       url: articleUrl,
       media: videoUrl,
       thumbnail: thumbnail,
       duration: duration,
       format: detectFormat(videoUrl), // "youtube", "mp4", etc.
       type: "source_video"
     });
   }
   ```

3. **Prioritization Logic:**
   - Video sources take priority over images
   - If multiple videos found, select the longest (likely main content)
   - Store ALL media, but mark primary with `"primary": true` flag

**Example Implementation (Node.js):**
```javascript
const ytdl = require('ytdl-core');
const cheerio = require('cheerio');

async function extractVideo(articleHtml, articleUrl) {
  const $ = cheerio.load(articleHtml);
  
  // Check for YouTube embeds
  const youtubeIframe = $('iframe[src*="youtube.com"], iframe[src*="youtu.be"]').first();
  if (youtubeIframe.length) {
    const youtubeUrl = youtubeIframe.attr('src');
    const videoId = extractYoutubeId(youtubeUrl);
    const info = await ytdl.getInfo(videoId);
    
    return {
      source: articleUrl,
      url: articleUrl,
      media: `https://www.youtube.com/watch?v=${videoId}`,
      thumbnail: info.videoDetails.thumbnails[0].url,
      duration: parseInt(info.videoDetails.lengthSeconds),
      format: "youtube",
      type: "source_video",
      primary: true
    };
  }
  
  // Check for HTML5 video
  const videoTag = $('video source[src]').first();
  if (videoTag.length) {
    const videoUrl = videoTag.attr('src');
    // Note: Duration requires downloading video or using HEAD request
    return {
      source: articleUrl,
      url: articleUrl,
      media: videoUrl,
      format: videoUrl.endsWith('.webm') ? 'webm' : 'mp4',
      type: "source_video",
      primary: true
    };
  }
  
  // Fallback to images (existing logic)
  return extractImages($, articleUrl);
}
```

## Frontend Integration Points

### Current Frontend Logic:
```tsx
// App.tsx - TV Mode filtering
const tvItems = liveNews.map(n => {
  const hasVideo = n.media && n.media.some(m =>
    getYoutubeId(m.url) ||
    getYoutubeId(m.media) ||
    m.media?.toLowerCase().endsWith('.mp4') ||
    m.media?.toLowerCase().endsWith('.webm')
  );
  return { ...n, hasVideo };
});
```

### With New Schema:
```tsx
const tvItems = liveNews.map(n => {
  const hasVideo = n.media && n.media.some(m =>
    m.format === 'youtube' ||
    m.format === 'mp4' ||
    m.format === 'webm' ||
    m.type === 'ai_generated'
  );
  return { ...n, hasVideo };
});
```

**Benefits:**
- More accurate video detection (no regex needed)
- Supports future AI-generated content
- Cleaner code, easier maintenance

## Migration Strategy

### Backward Compatibility
The frontend is designed to handle both old and new schema formats:

**Old format (no `format` or `type`):**
```json
{"media": "https://example.com/video.mp4"}
```
✅ **Still works** - Frontend uses file extension detection

**New format:**
```json
{"media": "https://example.com/video.mp4", "format": "mp4", "type": "source_video"}
```
✅ **Preferred** - Frontend uses explicit fields

2. **Week 2:** Monitor new events, verify data quality
3. **Week 3:** Backfill top 100 most-viewed events with video metadata
4. **Week 4:** Full production deployment

## API Rate Limits & Optimization

### YouTube Data API (for metadata)
- **Free Tier:** 10,000 quota units/day
- **Cost per request:** 1 unit (for `videos.list`)
- **Recommendation:** Cache video metadata in local database

### Video Duration Extraction (yt-dlp)
- **No rate limits** (uses public APIs)
- **Recommendation:** Run async, timeout after 30 seconds

### Example Caching Strategy:
```javascript
// Cache YouTube metadata for 7 days
const cacheKey = `yt:${videoId}`;
const cached = await redis.get(cacheKey);
if (cached) return JSON.parse(cached);

const info = await ytdl.getInfo(videoId);
await redis.setex(cacheKey, 604800, JSON.stringify(info)); // 7 days
return info;
```
